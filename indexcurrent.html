CIA Triad : Confidentiality

Confidentiality has to do with keeping an organization’s data private. This means that only authorized users and processes should be able to access or modify data.

×

CIA Triad : Integrity

Integrity means that data can be trusted. It should be maintained in a correct state, kept so that it may not be tampered with, and should be correct, authentic, and reliable.

×

CIA Triad : Availability

Just as it is important that unauthorized users are kept out of an organization’s data, data should be available to authorized users whenever they require it. This means keeping systems, networks, and devices up and running.

Rogue security software, or scareware, fools victims into installing malware, divulging confidential information. Rogue AV software mimics the look and feel of legitimate security software programs. Once installed, it claims to detect a large number of nonexistent threats so users will pay for the full version of the software.

TLS

TLS (Transport Layer Security), is the successor of SSL. It is a cryptographic protocol that provides security over a computer network. It is used in applications like email, instant messaging, and voice over IP, and securing HTTPS. The TLS protocol supports cryptography through the use of certificates, between communicating computer applications. It is itself composed of the TLS record and the TLS handshake protocols.

×

SSL

SSL (Secure Sockets Layer) is a security technology for establishing an encrypted link between a server(like a website) and a client (like a browser), or a mail server and a mail client (like Outlook). SSL is now deprecated, and has been succeeded by TLS.

×

SSH

The SSH (Secure Shell) Protocol is a cryptographic network protocol for operating network services securely over an unsecured network. It handles remote login and command-line execution. SSH applications connect an SSH client with an SSH server.

×

Encryption

Encryption transforms a message from easily read text, to an encoded format that cannot be read. The message is converted to an intelligible form by reversing the encryption.

×

Decryption

This is the process that uses an algorithm, and perhaps also a key, to reverse the process of encryption and restore an encrypted message to a form that can be read again.

×

Ciphertext

This is encrypted text, for example Harry Kane might appear as Ibssz Lbof after being encrypted using a Caesar Cipher with an alphabetic shift of 1 character.

×

Plaintext

This is decrypted text, plain text. For example, encrypted text, Ibssz Lbof might appear as Harry Kane after being decrypted using an alphabetic shift of 1 character.

×

Cryptographic Algorithms

This is a series of steps for transforming plaintext into an unintelligle form, called cipthertext. For instance, using an alphabetic shift of 2 letters to the right, IUPUI becomes KWUWK.

×

Cryptographic Key

This is part of the strategy used to encrypt plaintext. It could an alphabetic shift, a sequence of characters or a number used to convert plain text into unintelligible (without the key!) form.

×

Rainbow Table

A listing of all possible plaintext permutations of encrypted passwords - derived by accumulating data about an individual.

Rainbow tables are used by password cracking software. Systems with password-based authentication store databases of passwords for user accounts, typically encrypted rather than plaintext as a security measure. Once an attacker gains access to a password database, The rainbow table associates plaintext possibilities with each of those hashes.

×

Digital Certificate

A file or password that proves the authenticity of a device, server, or user through the use of cryptography. Commonly used to confirm the authenticity of a website to a web browser, which is also known as an SSL or TLS certificate. Digital certificates contain a copy of a public key from the certificate holder, which is verified by matching to a corresponding private key. A public key certificate is issued by certificate authorities (CAs), to verify identity.

Two-factor Authentication

A password alone may not keep an account secure, so two-factor authentication has become the norm. Besides a login username and password, two-factor authentication requires a second step, like verification of a security question or receipt of an e-mail.

Strong Passwords

A strong password has:

  • at least 15 characters
  • uppercase and lowercase letters
  • symbols, like ` ! ” ? $ ? % ^ & * ( ) _ – + = { [ } ] : ; @ ‘ ~ # | < ,> . ? /

A strong password is not a username, name, date of birth, dictionary word.

×

Password Entropy

Total guesses needed to guess a password. Passwords with higher entropy are more secure.

Calculate entropy, using L – The required length of the password R – The required range of characters For passwords with only lower case letters, R = 26. If the password must contain lower, upper and digits, R = 62 (26 + 26 + 10)

RL is large, so entropy is defined as the binary log of the value.

entropy (E): E = log2(RL)

Password Manager

A password manager is an encrypted digital vault that stores secure password login information for accounts. In addition to keeping your identity, credentials and sensitive data safe, password managers generate strong, unique passwords. Passward managers are available as apps and browser extensions.

×

Brute Force Attack

A brute force attack, also known as an exhaustive search, is a cryptographic hack that relies on guessing possible combinations of a targeted password until the correct password is discovered. The longer the password, the more combinations to be tested.

×

Dictionary Attack

A password dictionary attack is a hacking method used to break into a password-protected computer or server by entering every word in a dictionary as a password.

×

Malware

Malware is any software designed to steal information or cause harm to a computer system or an individual.

×

Rainbow Table Attack

A password cracking method that uses a special table to crack the password hashes in a database. A special set of data, the rainbow table, contains the password hashes for a single password or several passwords. If hackers gain have an accurate list of password hashes, they can crack all passwords quickly with a rainbow table.

×

Payload

The component of the attack which causes harm. Like the Greek soldiers hiding inside the Trojan horse, a malicious payload can sit harmlessly until triggered. Attack vectors can contain one or more malicious payloads.

×

Virus

A computer virus is a program that replicates itself by modifying other computer programs and inserting its own code (code injection). If this replication succeeds, the affected areas are then said to be "infected" with the virus, a metaphor derived from biological viruses!

×

Root-Kit

Rootkits remain hidden on a computer while remaining active. Rootkits allow hackers to - steal passwords and online private information - disable security software, making them especially hard to detect.

×

Worm

Type of malware that self-replicates and infects other computers while remaining active on infected systems. A worm duplicates to spread to uninfected computers. It does this by exploiting weaknesses in the operating system. It is common for worms to be noticed only when their uncontrolled replication consumes system resources, slowing or halting other tasks.

×

Trojan

The term Trojan comes from the Greek Trojan horse, which led to the fall of the city of Troy. This virus hides within seemingly harmless programs or downloads. Trojans don’t self-replicate. Instead, a Trojan horse spreads by pretending to be useful software or content while secretly containing malicious instructions.

×

Dropper

A type of Trojan. Purpose is to install other malware - they drop malware into a compromised system. Droppers are the malware that precipitates malware.

×

Ransomware

Malicious software that blocks access to a computer system until a sum of money is paid by the victim.

A botnet is a network of computers infected by malware, controlled by a bot-herder.Each machine controlled by the bot-herder is known as a bot.

Firewall

Software or hardware that monitors attempts to gain access to an operating system, and blocks unwanted traffic.

A firewall acts as a filter between a computer and a network such as the internet. It helps to protect information by managing network traffic - blocking unsolicited incoming network traffic and validating access by assessing network traffic for malware. Operating systems usually come with a pre-installed firewall.

Home routers use network address translation (NAT) to share a single IP address from the ISP among computers in your house. A NAT firewall is able to assess internet traffic and block unsolicited communications. It only accepts web traffic if a device on the house network solicited it.

Open Ports

Every IP address contains two kinds of ports, TCP and UDP, and there can be up to 65,535 of each for any given IP address. Services that connect to the internet (like web browsers, email clients, and file transfer services).

Any internet-connected service requires specific ports to be open in order to function. These can be exploited by malware, cybercriminals use open ports to access a computer. Close any ports that aren’t associated with a known legitimate service!

Cybercriminals can get a valid SSL certificate, and encrypt malicious content so it is undetectable. SSL Inspection inspects encrypted traffic to verify authenticity. It adds time to communications, but also a measure of safety.

Man In The Middle

In man-in-the-middle(MITM, MIM) attacks, the man in the middle, intercepts a victim’s communications before it is relayed.

A cybercriminal gains access to a poorly secured Wi-Fi router, like free Wi-Fi hotspots. The attacker inserts software between the victim’s computer and websites the user visits to capture login and other personal information.

An evil twin is a fake Wi-Fi access point that appears legitimate and is set up to eavesdrop on wireless communications. The attacker snoops on Internet traffic using the bogus wireless access point. Web users are invited to log into the attacker's server, and enter sensitive information such as usernames and passwords.

IMSI Catcher

An international mobile subscriber identity-catcher, or IMSI-catcher, is a telephone eavesdropping device used for intercepting mobile phone traffic and tracking location data of mobile phone users.

The device poses as a fake mobile tower between the target mobile phone and the service provider's real towers. The 3G and the LTE wireless standards require authentication from both the handset and the network, but attacks may be able to downgrade 3G and LTE to non-LTE network services which do not require mutual authentication.
